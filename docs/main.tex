\section{Introdução}
Este projeto consiste no desenvolvimento de um sistema desktop para gerenciamento de torneios de futebol. O foco principal é a estruturação de um banco de dados relacional robusto capaz de armazenar dados de equipes, atletas, partidas e eventos (gols e cartões), permitindo a geração automática de tabelas de classificação e estatísticas de desempenho.

O diferencial técnico reside na integração de um motor de processamento em Python com uma interface gráfica moderna e reativa, garantindo integridade referencial e eficiência nas consultas ao banco de dados.

\section{Objetivos}
\begin{itemize}
    \item \textbf{Modelagem de Dados:} Criar um esquema relacional normalizado para gerenciar a complexidade de um torneio esportivo.
    \item \textbf{Automação de Cálculos:} Implementar lógica de negócio para processamento de rankings e critérios de desempate via código.
    \item \textbf{Gestão Administrativa:} Fornecer uma interface segura para que administradores realizem a manutenção dos dados (CRUD).
\end{itemize}

\section{Requisitos do Sistema}

\subsection{Requisitos Funcionais}
\begin{itemize}
    \item Cadastro e gerenciamento de times e jogadores.
    \item Registro de partidas com definição de mandantes e visitantes.
    \item Súmula digital para registro de eventos (gols, cartões) em tempo real.
    \item Cálculo dinâmico da tabela de classificação (vitórias, empates, derrotas e saldo de gols).
    \item Sistema de autenticação para diferenciar administradores de visitantes.
\end{itemize}

\subsection{Requisitos Não Funcionais}
\begin{itemize}
    \item \textbf{Persistência:} Uso do PostgreSQL para armazenamento seguro.
    \item \textbf{Interface:} Aplicação nativa Desktop com visual moderno (Material Design 3).
    \item \textbf{Segurança:} Criptografia de dados sensíveis e controle de permissões.
\end{itemize}

\section{Modelo Conceitual}
O Modelo Entidade-Relacionamento (ER) foca na relação entre entidades como Equipes, Jogadores e Partidas, garantindo que cada evento de jogo esteja vinculado ao contexto correto.

\section{Tecnologias Utilizadas}

\subsection{Linguagem e Frameworks}
\begin{itemize}
    \item \textbf{Python 3.x:} Linguagem base para o motor do sistema.
    \item \textbf{Flet:} Framework baseado em Flutter para o desenvolvimento da interface gráfica (GUI) nativa.
    \item \textbf{SQLAlchemy:} ORM para mapeamento objeto-relacional e manipulação do banco de dados.
\end{itemize}

\subsection{Banco de Dados}
\textbf{PostgreSQL:} SGBD Relacional escolhido pela robustez e suporte a consultas complexas e integridade referencial.

\section{Arquitetura do Sistema}
A aplicação segue uma estrutura modular para separar a lógica de banco de dados da interface do usuário.

\begin{lstlisting}[basicstyle=\ttfamily\small, frame=single]
torneio_flet/
├── assets/           # Logos e recursos visuais
├── src/
│   ├── database/     # Modelos SQLAlchemy e Conexao
│   ├── services/     # Regras de Negocio (Ranking/Partidas)
│   ├── ui/           # Views Flet e Componentes
│   └── config.yaml   # Configuracoes do sistema
├── main.py           # Entrada da aplicacao
└── requirements.txt  # Dependencias
\end{lstlisting}

\section{Conclusão}
A implementação proposta une a teoria de bancos de dados relacionais com práticas modernas de desenvolvimento de software. A modularização permite que o sistema seja escalável, facilitando futuras expansões para outras modalidades esportivas ou integração com servidores remotos.
